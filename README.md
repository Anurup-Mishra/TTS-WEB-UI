# 🎙️ Full-Stack AI Voice & Audiobook Generator

A powerful, full-stack web application built with Flask that synthesizes text to speech using state-of-the-art neural models (Coqui XTTS v2). This project features zero-shot voice cloning, background music mixing, multilingual translation, and a fully integrated cloud backend using Supabase.

![Python](https://img.shields.io/badge/Python-3.9+-blue.svg)
![Flask](https://img.shields.io/badge/Flask-Web%20Framework-lightgrey.svg)
![Supabase](https://img.shields.io/badge/Supabase-Database%20%26%20Storage-47C273.svg)

## ✨ Key Features

* 🗣️ **Zero-Shot Voice Cloning:** Upload a short `.wav` sample of any voice (or choose from built-in celebrity presets) to instantly clone it.
* ☁️ **Cloud Database & Storage:** Automatically uploads generated MP3 files to a Supabase Storage bucket and logs all generation history into a PostgreSQL database.
* 🎵 **Audio Mixing:** Upload background music to seamlessly mix underneath the generated AI speech using `pydub`.
* 🌐 **Multilingual Support:** Type in English, and the app can auto-translate and speak in multiple different languages.
* ⚡ **Smart Fallbacks:** Uses Coqui TTS for high-quality neural generation, with an automatic fallback to `pyttsx3` or `gTTS` depending on the use case.

## 🛠️ Tech Stack

* **Backend:** Python, Flask
* **AI/TTS Engine:** Coqui TTS (XTTS_v2), PyTorch, pyttsx3, gTTS
* **Audio Processing:** FFmpeg, Pydub
* **Cloud Infrastructure:** Supabase (PostgreSQL + Object Storage)
* **Frontend:** HTML5, CSS3, Jinja2 Templates

## 🚀 Getting Started

### Prerequisites
1.  **Python 3.9+** installed on your system.
2.  **FFmpeg** installed and added to your system PATH.
3.  A **Supabase** account (Free tier is fine).

### 1. Clone the Repository
```bash
git clone [https://github.com/Anurup-Mishra/TTS-WEB-UI.git](https://github.com/Anurup-Mishra/TTS-WEB-UI.git)
cd TTS-WEB-UI
2. Set Up the Virtual Environment
Bash
python -m venv .venv
# On Windows:
.venv\Scripts\activate
# On Mac/Linux:
source .venv/bin/activate
3. Install Dependencies
Bash
pip install -r requirements.txt

# Note: Coqui TTS (XTTS_v2) works best with specific versions of torch and transformers.
# If you encounter loading errors, ensure these specific versions are installed:
pip install torch==2.5.1 torchvision==0.20.1 torchaudio==2.5.1
pip install transformers==4.38.2
4. Supabase Cloud Configuration
Create a Supabase project.

Create a public Storage bucket named audio_files.

Create a database table using the SQL Editor:

SQL
create table public.history (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()),
  text text not null,
  language text not null,
  audio_url text not null
);
Create a .env file in the root directory and add your project credentials (use the service_role key to allow backend uploads):

Code snippet
FLASK_SECRET=your_super_secret_flask_key
SUPABASE_URL=[https://your-project-id.supabase.co](https://your-project-id.supabase.co)
SUPABASE_KEY=your_service_role_secret_key
5. Run the Application
Bash
flask run
Open your browser and navigate to http://127.0.0.1:5000.

Note: The first time you synthesize a voice using the "Original" or "Cloned" options, the app will download the ~2GB XTTS model to your machine. Please be patient while it loads into memory!

📁 Directory Structure
app.py: Main Flask application and routing logic.

tts/synth.py: Core AI synthesis manager, model loading, and Supabase integration.

tts/audio_utils.py: Pydub logic for mixing voice with background music.

voices/: Directory to store custom .wav voice cloning presets.
